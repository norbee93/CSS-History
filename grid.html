G r i d

CSS Grid Layout is useful for dividing a page into areas based on column and row lines. Like Flexbox, a Grid container is defined in CSS, and all elements inside the container become Grid elements that can be positioned based on the Grid lines. 1

According to Bert Bos, he had thought about grid-based web layouts for CSS, because he wanted to capture elements of book and magazine layouts. 2 The W3C had received grid-based layout proposals before 2011, but this was the first that had implementation to consider. The 2011 Grid proposal includes comparison to tables, in the way that elements can be aligned with rows and columns but allows for more variety of layouts that are not possible with table based layout. Grid is designed to be easier than table or float based layouts, to adapt layouts to available space. It also offers a more ideal way of overlapping elements, where position is determined by grid lines. It was previously difficult to overlap elements in a flexible way. 3

In the 2013 draft of Grid, it was included that Grid was intended to be a two-dimensional system. This differed from Flexbox, in which 2 dimensional layout was only possible by nesting flex container boxes. 4 Rachel Andrew, a web developer and Invited Expert of the CSS Working Group, 5 was an early experimenter with CSS, after she attended a lecture about grid-based layouts by Bert Bos. She made examples to help people learn CSS Grid Layout.
