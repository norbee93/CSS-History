CSS History

When the first website was made in 1991 by Tim Berners Lee, there was no styling of the content, it was just HTML. 1 There were different ideas about how websites should introduce styling elements. One idea was using HTML to code style elements. Netscape was one browser that implemented presentational elements directly in the HTML. Another idea was to have a stylesheet associated with a certain browser. Pei-Yuan Wei developed Viola in 1992, 2 a browser with its own built in style sheet language. 3

There also came a push for the styling of the HTML elements to be seperate from the HTML itself. For example, Robert Raisch made a proposal in 1993 to create a separate format to hold styling information,

“There is a need within the WWW to be able to specify rendering specific information along with the content tagging in a WWW document. It is not appropriate to use HTML for this purpose, since one of the first principles of HTML is to encode objects within a document, not how they might be rendered in a particular environment.” -Robert Raisch4

Among Raisch and Wei there were proposals made for other style sheets before CSS.5
In October of 1994 Tim Berners-Lee founded the World Wide Web Consortium(W3C). 6 The W3C played in an important role in standardizing web languages and practices. Around the same time in 1994, Hakon W. Lie proposed a rough draft of CSS called Cascading HTML Style Sheets. 7 Bert Bos responded to Hakon W. Lie’s proposal. Bert Bos was working on an another styling language for the browser Argo, called Stream-based Style Sheet Proposal or (SSP).

In 1995 Hakon W Lie and Bert Bos released Cascading Style Sheets Level 1 8 , which allowed the style sheet authors to change the presentation of HTML files with fonts colors and layout. With the W3C gaining popularity by companies that wanted to comply to its' standards, browsers began to implement CSS recommendations. Microsoft released Internet Explorer 3.0 in 1996, which was the first widely used browser with CSS support. 9 Other browsers such as Netscape and Opera followed shortly after. 10 Drafts for the next CSS specification, CSS2 began in 1997 and stated that the editors wanted to consider both 'CSS authors and CSS implementers' 11

In 2000 Internet Explorer 5 for Macintosh was released by Microsoft. 12 It was the default installed browser for Mac computers, since this was before Safari, but the significance was that it took many of the W3C recommendations for CSS. This was a big step for standardizing web practices, but how different browsers implemented CSS caused bugs and made things difficult on developers. They made hacks and workarounds to make CSS work how they wanted. An example of this was the CSS box model. The box model helps specify how the HTML elements are spaced and displayed on the page. Internet explorer had its own box model that differed from the W3C model in the way it calculated the total width of elements with padding and margins. 13 Many preferred Internet Explorer’s version, because it counted the margin and padding space around the element as part of the total width of the content. Though W3C standardized its own box model version in 2001, 14 some designers were still using work-arounds to get IE's version in the 2010s. 15
